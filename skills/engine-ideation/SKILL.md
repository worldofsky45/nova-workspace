# Engine Ideation Specialist

Generate ideas for self-improving, data-driven engines that create real revenue opportunities.

## Mission

Identify niche markets where a **predictive + self-improving engine** can:
1. Make better decisions than humans
2. Learn from outcomes automatically
3. Compound intelligence monthly
4. Generate measurable ROI

## What Makes a Good Engine Idea

### Must Have (Non-Negotiable)
- ✅ **Clear feedback loop** - Can measure right/wrong objectively
- ✅ **Frequent iterations** - Daily/weekly decisions, not yearly
- ✅ **Quantifiable outcomes** - $ made, time saved, accuracy %
- ✅ **Self-improvement possible** - Patterns emerge from data
- ✅ **Automation potential** - Can run without constant human input

### Nice to Have
- 🎯 **Arbitrage opportunity** - Information asymmetry you can exploit
- 🎯 **First-mover advantage** - Market not yet saturated with AI
- 🎯 **Scalable** - Success in one area → replicate elsewhere
- 🎯 **Low capital requirements** - Can start with <$1,000
- 🎯 **Fast validation** - Know if it works within weeks, not months

### Red Flags (Avoid)
- ❌ Outcomes take >3 months to measure
- ❌ Success depends on factors you can't control
- ❌ No clear way to track performance
- ❌ Requires large upfront capital
- ❌ Market already dominated by better-funded competitors

## Engine Template (Reusable Pattern)

Every successful engine has these components:

### 1. Data Collection Layer
**Purpose:** Gather signals from multiple sources
**Pattern:** API calls + web scraping + RSS feeds + cached data
**Quality:** Retry logic, circuit breakers, fallback chains

### 2. Analysis Layer
**Purpose:** Process data → generate predictions/signals
**Pattern:** Pattern detection, anomaly spotting, trend analysis
**Quality:** Confidence levels, risk/reward scoring

### 3. Execution Layer
**Purpose:** Act on signals (auto or alert for manual)
**Pattern:** Position sizing, risk management, staged entries/exits
**Quality:** Stop-losses, validation checks, safety limits

### 4. Tracking Layer
**Purpose:** Record every decision + outcome
**Pattern:** JSONL logs, scorecard.jsonl, trades.jsonl
**Quality:** Timestamped, structured, immutable append-only

### 5. Learning Layer
**Purpose:** Analyze outcomes → improve future decisions
**Pattern:** Weekly calibration, monthly strategy updates
**Quality:** Statistical significance, pattern confirmation, A/B testing

### 6. Alert Layer
**Purpose:** Surface opportunities + warn of failures
**Pattern:** Telegram alerts, dashboard updates, emergency notifications
**Quality:** Actionable, not noisy, priority-based

## Ideation Framework

When generating new engine ideas, use this checklist:

### Step 1: Identify the Opportunity
- What market/niche has information asymmetry?
- Where do humans make slow/emotional/biased decisions?
- What data is publicly available but under-utilized?
- Where can speed + consistency beat expertise?

### Step 2: Define the Prediction
- What exactly are we predicting?
- How do we measure right vs. wrong?
- How quickly do we get feedback?
- What's the base rate (random guess accuracy)?

### Step 3: Map the Data Sources
- What free/cheap APIs exist?
- What can be scraped legally?
- What proxies exist for direct signals?
- How reliable are these sources?

### Step 4: Design the Feedback Loop
- How often do we make predictions? (Daily? Hourly?)
- How long until we know if we were right?
- What's logged? (Entry, outcome, confidence, reasoning)
- How do we calculate performance? (Win rate, ROI, accuracy)

### Step 5: Sketch the Self-Improvement
- Weekly: What patterns emerge from 7 days of data?
- Monthly: What strategies get better/worse over time?
- What thresholds can be tuned? (Confidence, timing, sizing)
- What can be A/B tested?

### Step 6: Estimate ROI Potential
- Best case: What if it works really well?
- Base case: What if it's decent?
- Worst case: How much do we lose if it fails?
- Time to break-even: How many iterations before profit?

## Example Engines (Reference Patterns)

### 1. Crypto Intel (Built, Active)
- **Predicts:** BTC/ETH/altcoin price movements
- **Feedback:** Target hit / stop-loss hit (hours to days)
- **Data:** CoinGecko, DeFiLlama, Fear & Greed, on-chain metrics
- **Learning:** Win rate by conviction level, asset, pattern type
- **ROI Model:** Paper trading $1K → prove profitability in 13 days

### 2. Domain Flipper (Idea)
- **Predicts:** Which expiring domains will have resale value
- **Feedback:** Domain sells within 90 days (yes/no)
- **Data:** Expiring domain lists, SEO metrics (backlinks, DA/PA), brandability scores
- **Learning:** Which domain types sell, optimal pricing, best marketplaces
- **ROI Model:** Buy $10-50 domains, flip for $100-500 (2-10x returns)

### 3. Luxury Arbitrage (Idea)
- **Predicts:** Which luxury items are underpriced vs. market
- **Feedback:** Item sells above purchase price (within 30 days)
- **Data:** eBay completed listings, StockX prices, Grailed, Poshmark, authentication databases
- **Learning:** Best brands, condition multipliers, seasonal patterns, listing optimization
- **ROI Model:** Buy $200-500 items, flip for $300-800 (50-60% margins)

### 4. Real Estate Lead Scoring (Idea)
- **Predicts:** Which property leads convert to deals
- **Feedback:** Lead converts to closed deal (30-90 days)
- **Data:** Zillow, Redfin, PropStream, county records, MLS listings, seller motivation signals
- **Learning:** Best neighborhoods, ideal price points, time-on-market sweet spots, seller profiles
- **ROI Model:** Score 1000 leads → identify top 50 → close 5 deals at $5K commission = $25K

### 5. Content Viral Predictor (Idea)
- **Predicts:** Which content ideas will hit >10K views
- **Feedback:** Post performance after 48 hours
- **Data:** Trending topics, sentiment analysis, historical performance, competitor content
- **Learning:** Best topics, optimal length, posting times, hooks that work
- **ROI Model:** 10 posts/week → 3 hit 10K+ views → audience growth → monetization

### 6. Airbnb Dynamic Pricing (Idea)
- **Predicts:** Optimal nightly rate to maximize occupancy + revenue
- **Feedback:** Booking rate, revenue per available night (daily)
- **Data:** AirDNA, local events, weather, competitor pricing, seasonality
- **Learning:** Price elasticity, best discounts, event premiums, ideal booking window
- **ROI Model:** Increase revenue 15-30% vs. static pricing

### 7. Sports Betting Value Finder (Idea)
- **Predicts:** Which bets have +EV (positive expected value)
- **Feedback:** Bet wins/loses (hours to days)
- **Data:** Odds from multiple books, team stats, injuries, weather, betting market movements
- **Learning:** Best bet types (spreads, totals, props), bankroll management, sharp vs. square lines
- **ROI Model:** 5% edge on 100 bets → 55% win rate → profit over time

### 8. Freelance Gig Arbitrage (Idea)
- **Predicts:** Which Upwork/Fiverr gigs can be outsourced profitably
- **Feedback:** Gig completed at profit margin (5-10 days)
- **Data:** Upwork API, Fiverr categories, overseas VA rates, completion times, client satisfaction
- **Learning:** Best niches, ideal markup %, client red flags, reliable VAs
- **ROI Model:** Charge $500, outsource for $100, net $400 per gig

### 9. Twitter/X Thread Optimizer (Idea)
- **Predicts:** Which thread angles will drive most engagement
- **Feedback:** Likes, retweets, impressions after 24 hours
- **Data:** Trending topics, competitor threads, historical performance, sentiment
- **Learning:** Best hooks, ideal length, posting times, virality patterns
- **ROI Model:** Audience growth → newsletter subs → product sales / sponsorships

### 10. Stock Market Earnings Plays (Idea)
- **Predicts:** Direction of stock after earnings (up/down/flat)
- **Feedback:** Stock price 24h after earnings release
- **Data:** Earnings whispers, options flow, analyst estimates, historical reactions, sentiment
- **Learning:** Which sectors predictable, guidance impact, beat/miss thresholds
- **ROI Model:** Options plays with defined risk, 60%+ win rate needed

## Output Format

When generating engine ideas, provide:

### 1. Opportunity Name
Short, memorable name (e.g., "Domain Flipper Engine")

### 2. One-Line Pitch
What it predicts and why it matters (e.g., "Predicts which expiring domains will sell for 5-10x returns within 90 days")

### 3. Core Prediction
- **What:** Exactly what are we predicting?
- **Feedback:** How do we know if we were right?
- **Timeline:** How long until we get feedback?

### 4. Data Sources (Free/Cheap)
- List 3-5 APIs, websites, or data feeds
- Note if any require paid plans
- Include fallback sources

### 5. Self-Improvement Mechanics
- **Weekly:** What gets analyzed after 7 days?
- **Monthly:** What strategies improve over time?
- **Thresholds:** What can be tuned?

### 6. ROI Model
- **Starting capital:** How much to start?
- **Per-iteration cost:** $ per prediction/action
- **Success rate needed:** What % accuracy to break even?
- **Upside:** Best case monthly revenue

### 7. Risk Assessment
- **Low risk:** What's the worst case loss?
- **Failure mode:** How do we know if it's not working?
- **Time to validation:** How quickly do we get proof?

### 8. Complexity Rating
- **Easy (1-2 weeks):** Mostly existing patterns, simple data
- **Medium (3-4 weeks):** New data sources, moderate complexity
- **Hard (1-2 months):** Custom ML, complex integrations

### 9. First Milestone
What's the MVP to prove the concept? (e.g., "Paper trade 20 domain predictions over 30 days, track sell-through rate")

## Execution Priorities

### Tier 1: Build Immediately (High Conviction)
- Clear feedback loop (daily/weekly)
- Free/cheap data sources available
- Low capital requirement (<$500)
- Fast validation (<30 days)
- Similar to proven patterns (crypto, content, etc.)

### Tier 2: Validate First (Medium Conviction)
- Feedback loop exists but slower (monthly)
- Some data sources cost money
- Medium capital ($500-2000)
- Moderate validation time (30-60 days)
- New territory, less proven

### Tier 3: Research Required (Low Conviction)
- Unclear feedback loop
- Data sources hard to access
- High capital requirement (>$2000)
- Long validation time (>60 days)
- Unproven market
